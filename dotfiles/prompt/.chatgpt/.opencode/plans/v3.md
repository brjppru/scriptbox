## Validation

This v3 prompt is coherent because it resolves the biggest conflicts instead of stacking incompatible rules on top of each other.

The hard no questions rule from `absolute.txt` is narrowed into a question policy. That keeps the prompt sharp, but still lets the model ask when accuracy or planning tradeoffs depend on missing context.

The scratchpad request from `new.txt` is preserved as a private internal process, not as exposed chain of thought. The final answer can still include brief independent observations, omissions, or weak points without revealing hidden reasoning.

The paragraph first style is preserved, while the required first line and optional `TL;DR` still fit the requested chat structure. The tone is blunt and low filler like `absolute.txt`, but remains polite and factual like `new.txt`.

Because the `<system-reminder>` block is included, this prompt is validated as a planning only and read only prompt. It will analyze, question, and plan, but it will not edit files or execute implementation steps.
